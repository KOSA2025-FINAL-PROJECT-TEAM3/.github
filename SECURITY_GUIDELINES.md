# 보안 및 문서 작성 가이드라인

## 개요

이 저장소는 조직 소개를 위한 **공개(Public) 레포지토리**입니다.
모든 문서 작성 시 보안을 최우선으로 고려해야 합니다.

## 보안 주의사항

### 민감 정보 보호

다음과 같은 민감 정보는 **절대 포함하지 않습니다**:

- 서버 접속 IP 주소
- 포트 번호
- API 키 및 인증 토큰
- 데이터베이스 접속 정보
- 내부 네트워크 구조
- 실제 운영 환경 설정값

### 정보 마스킹 방법

민감 정보를 예시로 사용해야 할 경우, 다음과 같이 마스킹합니다:

```
# ❌ 잘못된 예시
서버 주소: 192.168.1.100:8080
DB 접속: postgresql://user:password@10.0.0.5:5432/dbname

# ✅ 올바른 예시
서버 주소: <SERVER_IP>:<PORT>
DB 접속: postgresql://<USERNAME>:<PASSWORD>@<DB_HOST>:<DB_PORT>/<DB_NAME>
```

## 문서 작성 기준

### KISA Secure Coding 가이드라인 준수

모든 README 및 문서 작성 시 다음을 준수합니다:

1. **입력 데이터 검증 및 표현**
   - 코드 예제에서 사용자 입력값 검증 로직 포함
   - SQL Injection, XSS 등 취약점 예방 코드 제시

2. **보안 기능**
   - 인증/인가 관련 예제는 보안 모범 사례 기반으로 작성
   - 암호화 관련 정보는 안전한 알고리즘 사용 권장

3. **에러 처리**
   - 에러 메시지에 시스템 내부 정보 노출 금지
   - 스택 트레이스 등 디버깅 정보는 예제에서 제외

4. **코드 품질**
   - 안전한 코딩 패턴 사용
   - 보안 취약점이 없는 예제 코드 제공

## 기여 시 체크리스트

문서를 작성하거나 수정할 때 다음을 확인합니다:

- [ ] 민감한 정보가 포함되어 있지 않은가?
- [ ] 예제 코드에 보안 취약점은 없는가?
- [ ] IP 주소, 포트 등이 마스킹 처리되었는가?
- [ ] KISA Secure Coding 가이드라인을 준수하는가?
- [ ] 공개 저장소에 적합한 내용인가?
